# Story 2.2: 录入和覆盖当天健康记录

Status: done

## Story

As a 个人用户,
I want 录入或修改当天健康数据,
so that 每天只保留一条有效身体记录。

## Acceptance Criteria

1. Given 用户在受信浏览器进入“记录”页面, when 页面加载, then 系统显示中文健康记录表单。
2. 表单字段包含体重、腰围、臀围、体脂率，并显示中文标签和单位。
3. 保存时系统按当前本地日期创建当天健康记录。
4. 如果当天已有健康记录，系统更新原记录而不是新增第二条。
5. 非法数值在字段旁显示中文错误，并保留用户已输入内容。
6. 保存成功后显示中文成功提示，并在表单中显示最新保存值。
7. 该记录可被后续首页和历史页读取。
8. 未受信浏览器不能访问记录页面或提交保存入口。

## Dev Notes

### Implementation Notes

- `/records` 现在是受保护动态页面，读取当天健康记录并传给客户端表单。
- 表单状态常量放在普通模块，避免从 `"use server"` 模块向客户端导出普通值。
- 当天日期使用 `APP_TIME_ZONE`，默认 `Asia/Shanghai`；无效时区自动回退默认值。
- 保存入口先执行 `requireTrustedDevice()`，再解析表单并调用 records service。

# Story 1.1: 初始化应用骨架与中文布局

Status: done

## Story

As a 个人用户,
I want 打开瘦身助手时看到中文应用框架,
so that 我可以在统一的受保护界面中使用后续功能。

## Acceptance Criteria

1. Given 项目尚未有应用代码, when 开发者初始化应用, then 应用使用 Next.js、React、TypeScript、Tailwind、shadcn/ui 基础栈。
2. 页面显示名为“瘦身助手”。
3. 可见主界面文案为中文。
4. 桌面端具备左侧导航框架。
5. 移动端具备可适配的导航结构。
6. 不出现营销式首页或英文主界面。

## Dev Notes

### Scope

本故事只创建应用骨架和中文响应式布局。不要实现以下内容：

- 访问密码创建、设备信任、cookie/localStorage 校验。
- SQLite、Drizzle schema、repository。
- 健康记录、跑步记录、目标、提醒、SMTP。
- 真实图表、趋势计算、BMI 计算。

后续 Story 1.2 开始处理访问保护数据模型，Story 1.3/1.4 处理访问流程。

### Repository State

当前仓库不是空目录，但没有应用代码：

- 已有：`README.md`、`LICENSE`、`doc/`、`docs/`、`_bmad/`、`_bmad-output/`、`.gitignore`。
- 未发现：`package.json`、`app/`、`components/`、`features/`、`lib/`。

实现时要在仓库根目录落地应用文件。若使用 `create-next-app`，应先在临时目录生成，再把必要应用文件合并回根目录，避免覆盖现有规划文档。

### Architecture Compliance

必须遵守架构 Spine：

- 使用分层功能切片的模块化单体。
- `app/` 只负责路由、页面组合和调用未来 server actions / route handlers。
- `features/*/components` 放功能组件；`components/layout` 放全局布局；`components/ui` 放 shadcn/ui 基础组件。
- 可见 UI 文案使用中文，显示名为“瘦身助手”。
- UI 必须继承 `EXPERIENCE.md` 和 `DESIGN.md`；`mockups/key-screens.html` 作为布局参考。
- MVP 目标是长运行 Node.js 进程，不面向 serverless/edge 初始化。

Suggested initial structure:

```text
app/
  layout.tsx
  page.tsx
  globals.css
components/
  layout/
    app-shell.tsx
    desktop-sidebar.tsx
    mobile-nav.tsx
  ui/
features/
  access/
    components/
  dashboard/
    components/
lib/
  cn.ts
```

不要在本故事创建 `db/`、`integrations/` 或持久化实现，除非脚手架工具创建了无业务含义的基础配置文件。

### UX Requirements

首屏必须像个人健康控制台，而不是宣传页：

- 桌面：左侧常驻导航、内容区域最大宽度约 1200px。
- 移动：顶部标题栏、底部导航，点击目标至少 44px 高。
- 首页主操作是“记录今天”，但该按钮可暂时链接到占位状态或禁用说明。
- 首页可以展示占位卡：今日状态、健康摘要、运动摘要、目标进度。占位内容必须明确是“暂无数据”或“待记录”，不要伪造真实数据。
- 颜色不能作为唯一信息来源；状态旁要有中文文字。
- 字体使用 Inter 或系统无衬线字体；不要使用负字距。

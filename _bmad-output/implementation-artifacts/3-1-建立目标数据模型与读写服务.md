# Story 3.1: 建立目标数据模型与读写服务

Status: done

## Story

As a 个人用户,
I want 我的健康目标和跑步目标可以被系统保存和读取,
so that 首页、目标页和后续反馈都能基于同一套目标数据。

## Acceptance Criteria

1. Given 用户需要设置健康目标和跑步目标, when 数据库 schema 扩展, then 系统创建目标数据结构，按目标类型区分健康目标和跑步目标。
2. 健康目标支持目标体重、目标腰围、目标臀围、目标体脂率，其中除目标体重外允许为空。
3. 跑步目标支持每周跑步次数和每周跑量。
4. 目标记录使用字符串 UUID 主键。
5. repository/service 提供读取、创建、更新目标的基础方法。
6. UI、server action、route handler 不直接写数据库。

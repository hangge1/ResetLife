# Story 3.4: 构建首页今日状态和指标卡

Status: done

## Story

As a 个人用户,
I want 在首页快速看到今天是否完成记录和关键指标,
so that 我不用进入多个页面就能判断今天该做什么。

## Acceptance Criteria

1. Given 用户在受信浏览器进入首页, when 系统读取当天健康记录、当天跑步记录和当前目标, then 首页显示今日健康记录状态。
2. 首页显示今日跑步记录状态。
3. 首页显示当前提醒状态；未配置提醒时显示“未开启提醒”或等价中文状态。
4. 每个状态项可以跳转到对应操作页面。
5. 首页显示健康摘要和运动摘要指标卡。
6. 指标卡包含名称、数值、单位和中文变化说明。
7. 状态和指标含义不只依赖颜色表达。
8. 空数据时显示中文空状态或引导设置/记录。

## Dev Notes

- 本故事只做今日状态和静态摘要，不做趋势图、BMI、目标差距、预计达成时间和鼓励文案。
- 首页应复用 `requireTrustedDevice()`。
- read-model service 可以组合 records 和 goals repository 的读取方法，但 UI 不直接写数据库。
- 中文单位沿用内部单位：公斤、厘米、公里、次。

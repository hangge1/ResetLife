# Story 4.1: 建立配置中心基础与设置数据模型

Status: done

## Story

As a 个人用户,
I want 在一个配置中心维护应用参数,
so that 个人资料、提醒、SMTP 和估算规则不会散落在不同位置。

## Acceptance Criteria

1. Given 应用需要保存多类用户可配置参数, when 数据库 schema 扩展, then 系统创建 typed settings 数据结构，支持按 key/type 保存单例配置。
2. repository/service 提供读取和保存设置的基础方法。
3. 配置页面按个人资料、提醒、SMTP、趋势估算、访问保护分组展示。
4. 每个配置分组可以独立保存。
5. 表单显示中文标签、单位和字段错误。
6. UI、server action、scheduler 不直接写数据库。

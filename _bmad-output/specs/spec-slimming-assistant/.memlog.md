- (event) Create SPEC from final PRD, final UX, and final architecture spine. Express distill selected because inputs are complete and have no open questions.
- (capability) CAP-1: User can create or update daily health records and add running records so personal data is captured in one place; success is same-day health overwrite, multiple same-day run records, and recalculated dashboard values after save/edit/delete.
- (capability) CAP-2: User can set health and weekly running goals so progress has a target; success is dashboard shows current, target, remaining gap, and status for configured goals.
- (capability) CAP-3: System can present a Chinese dashboard for today status, health trends, running trends, BMI, goal progress, estimated time-to-goal, and data-based encouragement; success is user can understand 7-day and 30-day progress without opening raw tables.
- (capability) CAP-5: User can manage profile, SMTP, trend thresholds, access password, and trusted browser devices from settings; success is settings changes are used by reminders, dashboard estimates, and access protection without code or environment edits.
- (capability) CAP-4: User can configure reminders and SMTP email delivery so missed movement or recording is visible; success is in-app reminders and email reminders follow one reminder eligibility rule and do not duplicate for the same local date/type/channel.
- (capability) CAP-7: User can browse, filter, edit, and delete historical health and running records; success is changed records immediately affect trends, goals, and today status.
- (capability) CAP-6: System can protect personal health data with access password and trusted browser device identifiers, without user accounts; success is untrusted browsers cannot read/write personal data and trusted browsers can enter without retyping the password.
- (constraint) MVP is a single-user Chinese responsive web app named 瘦身助手; visible UI copy must be Chinese, clean, data-first, and follow DESIGN.md/EXPERIENCE.md.
- (constraint) Architecture must follow ARCHITECTURE-SPINE.md: modular monolith, Next.js/Node long-running process, local SQLite through Drizzle repositories, derived read models for BMI/trends/progress, ReminderRunner for reminders, no serverless/edge MVP deployment.
- (constraint) Data invariants are fixed: one health record per local date, multiple run records per local date, idempotent reminder events by localDate/type/channel, settings as typed singleton records, access password hash and trusted devices stored server-side.
- (constraint) MVP excludes multi-user accounts, public registration, social features, third-party data import, SMS/WeChat/QQ, AI coach, training prescriptions, diet prescriptions, medical diagnosis, and disease management.
- (decision) Overall success signal: user can use 瘦身助手 for 30 days instead of spreadsheets/notes, complete daily recording within one minute, and see truthful goal progress, trends, reminders, and data-based encouragement.

- (event) Self-validate pass 2 preservation: PASS. PRD capabilities, UX language/visual constraints, architecture ADs, data invariants, and non-goals are captured in SPEC.md or companions; wrapper-only upstream process details intentionally omitted.

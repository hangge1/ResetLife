# 术语表

- **健康记录** — 每日一次的身体数据记录，包括体重、腰围、腿围、体脂率等；同一 `localDate` 只能有一条。
- **跑步记录** — 一次跑步活动记录，包括运动时长、公里数、配速、平均心率、平均步幅、步频等；同一 `localDate` 可以有多条。
- **目标** — 用户设置的健康或跑步目标，例如目标体重、每周跑步次数、每周跑量。
- **目标进度** — 当前值、目标值、剩余差距、完成比例和状态判断。
- **趋势反馈** — 基于健康记录和跑步记录计算的 7 天 / 30 天变化方向、周期汇总和预计达成时间。
- **提醒** — 系统在缺少运动或记录时产生的站内提示或邮件提示。
- **配置界面** — 用户维护个人资料、SMTP、趋势估算阈值、访问密码和受信设备的页面。
- **BMI** — 由身高和体重计算的身体质量指数，仅用于辅助展示，不作为医疗判断。
- **访问密码** — 用于验证新浏览器访问权限的单一密码，服务端只保存哈希。
- **设备标识** — 受信浏览器验证通过后生成并保存在 cookie 或 localStorage 中的随机标识。
- **受信设备** — 服务端保存了设备标识的浏览器环境，后续访问不需要再次输入访问密码。
- **ReminderRunner** — 单一提醒决策运行器，负责判断提醒资格、生成站内提醒、触发 SMTP 邮件。

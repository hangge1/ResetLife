- (decision) AD-1 binds all MVP capabilities to a modular monolith with layered feature slices: Next.js app owns UI, server actions/route handlers own mutations, domain services own business rules, repositories own SQLite persistence. Prevents independently-built pages from bypassing domain rules or inventing separate backend surfaces.
- (decision) AD-2 binds persistence to one local SQLite database accessed only through Drizzle repositories. Prevents dashboard, records, goals, reminders, and settings from defining incompatible data shapes or direct SQL scattered across UI handlers.
- (decision) AD-3 binds computed dashboard values to read-model/query services, not stored mutable columns except source facts and settings. Prevents stale BMI, trend, goal progress, weekly aggregates, and estimated-time values after record edits/deletes.
- (decision) AD-4 binds access protection to server-local access password hash plus trusted browser device identifiers. Prevents accidental introduction of multi-user accounts, public registration, or client-only protection for personal health data.
- (decision) AD-5 binds reminders to an in-process ReminderRunner that reads reminder settings and latest records from SQLite, creates in-app reminders, and sends email through Nodemailer/SMTP when the app process is running. Prevents page-level reminders or separate ad-hoc schedulers with inconsistent eligibility logic.
- (decision) AD-6 binds configuration to typed settings records in SQLite: profile, SMTP, trend thresholds, reminders, access password metadata, trusted devices. Prevents environment-variable-only or UI-local configuration that cannot be managed from the Chinese settings UI.
- (decision) AD-7 binds UI implementation to the finalized Chinese UX spines: shadcn/ui + Tailwind tokens implement DESIGN.md; visible UI copy is Chinese; pages follow EXPERIENCE.md IA. Prevents English UI drift, marketing-style landing pages, and ad-hoc component styling.
- (decision) AD-8 binds deployment to one long-running Node.js process with local filesystem access to SQLite data and app-managed files. Prevents choosing serverless/edge deployment paths that break local SQLite, in-process reminders, or server-local password/trusted-device storage.
- (decision) AD-9 added after reviewer gate: core domain data invariants are fixed at architecture level: one health record per local date; multiple run records per local date; goals are typed records; reminder events are idempotent by local date/type/channel; settings are singleton typed records. Prevents module-level schema drift.

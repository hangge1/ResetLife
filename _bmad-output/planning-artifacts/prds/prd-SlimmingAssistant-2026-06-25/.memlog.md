- (event) Create PRD from doc/requirement.md and product brief brief-SlimmingAssistant-2026-06-25. Stakes calibrated as hobby/solo personal Web app feeding UX, architecture, and stories.
- (decision) PRD inherits confirmed MVP decisions: single personal user, manual entry, Web first, one health record per day with overwrite, goals for health and running, equal health/exercise dashboard priority, in-app and email reminders only.
- (event) Light reference scan: Strava/Garmin/MyFitnessPal emphasize goals, progress tracking, reports, and health/activity history; PRD should keep goal progress and trend feedback central rather than a plain logbook.
- (decision) Confirmed PRD decisions: v1 uses app-level single-user login without registration; email reminders use user-configurable SMTP settings in a configuration UI; trend-estimation thresholds are configurable with enforced lower bounds; BMI is shown using height; running records may have multiple entries per day while health records remain one per day with overwrite.
- (decision) Access protection changed from single-user login to device trust: first access from a device requires entering a server-local password; after verification the device unique ID is trusted and subsequent access from that device does not require re-entering the password. This is MVP lightweight protection, not a full account system.
- (decision) Trusted device identifier clarified: after password verification, browser generates and stores a random device identifier in cookie/localStorage; server stores it in the local trusted device list. Changing browser, clearing storage, private browsing, or using a new device requires password verification again.
- (decision) Remaining PRD open questions resolved by default: SMTP configuration supports None/SSL/STARTTLS with STARTTLS as default; estimated time-to-goal uses a simple linear trend in MVP, only when configured thresholds are met.
- (decision) Access password initialization resolved: if the server has no local access password, first access enters initialization mode to create the password; afterwards new browser environments must enter that password to become trusted devices.
- (event) PRD finalized: memlog decisions reconciled, source inputs reconciled, reviewer pass completed, no open questions or assumptions remain.
